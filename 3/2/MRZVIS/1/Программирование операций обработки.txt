Программирование операций обработки данных и знаний с конвейеризированной обработкой
Требуется:
1. построить модель арифметического (сбалансированного) конвейера,
реализующего операцию целочисленного деления по одному из 2-х алгоритмов либо - умножения по одному из вариантов алгоритма для р-разрядных чисел, удовлетворяющую нижеприведённым требованиям;
2. построить семейства графиков коэффициента ускорения и эффективности, фиксируя n и r;

Требования к модели арифметического конвейера:
1. модель должна обеспечивать визуальное отображение пошагового процесса умножения (деления)
(за один шаг происходит обработка данных на этапе конвейера и передача результатов на следующий этап); 2. для каждого этапа конвейера должны отображаться:
а. данные (включая промежуточные результаты) в двоичном формате;
б. номер (индекс) обрабатываемой пары элементов исходных векторов;
в. время, прошедшее с начала запуска конвейера;
3. для каждого этапа конвейера в конкретный момент времени должны отображаться входные,
либо выходные данные (на усмотрение разработчика);
4. перед первым этапом конвейера должно быть отражено в десятичном формате не менее
трех пар элементов исходных векторов со своими номерами (индексами).
5. после последнего этапа конвейера должно быть отражено в десятичном формате не менее
трёх элементов результирующего вектора С со своими номерами (индексами) и временной отметкой результата;
6. Время считать в условных временных единицах (например, тактах);
7. должна быть обеспечена возможность параметрического задания:
а. времени счёта ti на этапах сбалансированного конвейера;
б. параметра n;
в. параметра m.
8. желательный (но не обязательный), тип реализуемого конвейера - несбалансированный (в общем случае) конвейер с возможностью параметрического задания всех параметров: n, r, m, p. (ti};
9. двоичные данные отображать группами по четыре знака, отделяя группы пробелами, точкой или тире. Например:
1101 1010 0101
1101.1010.0101 1101-1010-0101
